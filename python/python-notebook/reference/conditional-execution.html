Alternative Execution

Alternative execution in Python refers to the ability to execute different sections of code depending on whether a certain condition is true or false. This is achieved using conditional statements such as if, else, and elif.

The basic structure of an if statement is as follows:

if condition:
    # code to execute if the condition is true

Here, the condition is a boolean expression that evaluates to either True or False. If the condition is true, the code within the indented block will be executed.

If you want to execute a different block of code when the condition is false, you can use the else statement:

if condition:
    # code to execute if the condition is true
else:
    # code to execute if the condition is false

If you have multiple conditions that you want to test, you can use the elif statement:

if condition1:
    # code to execute if condition1 is true
elif condition2:
    # code to execute if condition2 is true
else:
    # code to execute if both condition1 and condition2 are false

In this example, if condition1 is true, the code within the first indented block will be executed. If condition1 is false but condition2 is true, the code within the second indented block will be executed. If both conditions are false, the code within the else block will be executed.

Alternative execution is a fundamental concept in programming and is essential for making decisions in your code based on certain conditions. With practice, you'll get more comfortable using conditional statements and will be able to write more complex programs.

Chained Conditionals

Chained conditions in Python refer to the ability to test multiple conditions using logical operators like and and or in a single if statement. This allows you to check whether multiple conditions are true or false at once, and execute code accordingly.

For example, suppose you want to check whether a number is between 1 and 10, inclusive. You can do this using chained conditions as follows:

if 1 <= number <= 10:
    # code to execute if the condition is true

Here, the chained condition 1 <= number <= 10 checks whether the number variable is greater than or equal to 1 and less than or equal to 10. If the condition is true, the code within the indented block will be executed.

You can also use logical operators like and and or to combine multiple conditions. For example:

if age >= 18 and country == "USA":
    # code to execute if the condition is true

In this example, the chained condition age >= 18 and country == "USA" checks whether the age variable is greater than or equal to 18 and whether the country variable is equal to "USA". If both conditions are true, the code within the indented block will be executed.

Alternatively, you can use the or operator to create a condition where only one of the two conditions needs to be true:

if country == "USA" or country == "Canada":
    # code to execute if the condition is true

In this example, the chained condition country == "USA" or country == "Canada" checks whether the country variable is either "USA" or "Canada". If either condition is true, the code within the indented block will be executed.

Chained conditions allow you to write more concise and readable code, especially when you need to test multiple conditions at once.

Nested Conditionals

Nested conditionals in Python refer to the ability to include one or more conditional statements within another conditional statement. This allows you to test for more complex conditions and execute code based on the results of these tests.

Here are two important points:

  • this use of the if statement is known as nesting; remember that every else refers to the if which lies at the same indentation level; you need to know this to determine how the _if_s and _else_s pair up;
  • consider how the indentation improves readability, and makes the code easier to understand and trace.

For example, suppose you want to check whether a number is positive, negative, or zero. You can do this using nested conditionals as follows:

if number > 0:
    print("The number is positive.")
elif number < 0:
    print("The number is negative.")
else:
    print("The number is zero.")

In this example, the first conditional statement checks whether the number variable is greater than 0. If this is true, the code within the indented block is executed and the message "The number is positive." is printed.

If the first condition is false, the elif statement checks whether the number variable is less than 0. If this is true, the code within the second indented block is executed and the message "The number is negative." is printed.

If both the first and second conditions are false, the else statement is executed, and the message "The number is zero." is printed.

You can also include more complex conditions within nested conditionals, such as multiple conditions combined with logical operators:

if age >= 18:
    if country == "USA":
        print("You are eligible to vote.")
    elif country == "Canada":
        print("You are eligible to vote.")
    else:
        print("You are not eligible to vote.")
else:
    print("You are not old enough to vote.")

In this example, the first conditional statement checks whether the age variable is greater than or equal to 18. If this is true, the nested conditional statement checks whether the country variable is equal to "USA" or "Canada". If either condition is true, the code within the first indented block is executed and the message "You are eligible to vote." is printed.

If both conditions are false, the code within the second else block is executed, and the message "You are not eligible to vote." is printed.

If the first condition is false, the code within the second else block is executed, and the message "You are not old enough to vote." is printed.

Nested conditionals allow you to test for more complex conditions and execute code based on the results of these tests. However, be careful not to make your code too complex or difficult to read by nesting too many conditionals within one another.

if the_weather_is_good: 
	if nice_restaurant_is_found:
		have_lunch() 
	else: 
		eat_a_sandwich() 
else: 
	if tickets_are_available:
		go_to_the_theater() 
    else: 
    	go_shopping()

The way to assemble subsequent if-elif-else statements is sometimes called a cascade.

if the_weather_is_good:
    go_for_a_walk()
    
elif tickets_are_available:
    go_to_the_theater()
    
elif table_is_available:
    go_for_lunch()
    
else:
    play_chess_at_home()

Some additional attention has to be paid in this case:

  • you mustn't use else without a preceding if;
  • else is always the last branch of the cascade, regardless of whether you've used elif or not;
  • else is an optional part of the cascade, and may be omitted;
  • if there is an else branch in the cascade, only one of all the branches is executed;
  • if there is no else branch, it's possible that none of the available branches is executed.

Short Circuit Evaluation

When Python detects that there is nothing to be gained by evaluating the rest of a logical expression, it stops it's evaluation and does not do the computations in the rest of the logical expression. When the evaluation of a logical expression stops because the overall value is already known, it is called short-circuiting the evaluation.

While this may seem like a fine point the short circuit behaviour is next to a clever technique called the guardian pattern.

When evaluating a logical expression with the "and" operator, if the first operand evaluates to False, then the entire expression is False, and the second operand is not evaluated. Similarly, when evaluating a logical expression with the "or" operator, if the first operand evaluates to True, then the entire expression is True, and the second operand is not evaluated.

Here is an example of short circuit evaluation using the "and" operator:

x = 5
y = 10
if x > 0 and y/x > 2:
    print("y is greater than twice x")
else:
    print("y is not greater than twice x")

In this example, the second operand of the "and" operator (y/x > 2) is not evaluated if x > 0 is False, because the entire expression would be False regardless of the value of the second operand. This is because in an "and" expression, if any operand is False, the entire expression must be False.

Similarly, here is an example of short circuit evaluation using the "or" operator:

x = None
y = 10
if x is None or y/x > 2:
    print("y is either greater than twice x or x is None")
else:
    print("y is not greater than twice x and x is not None")

In this example, the second operand of the "or" operator (y/x > 2) is not evaluated if x is None is True, because the entire expression would be True regardless of the value of the second operand. This is because in an "or" expression, if any operand is True, the entire expression must be True.

Overall, short circuit evaluation can be a useful tool for improving the efficiency and readability of your code.

More on Conditional Execution

The first form of a conditional statement, which you can see below is written very informally but figuratively:

if true_or_not:
    do_this_if_true

This conditional statement consists of the following, strictly necessary, elements in this and this order only:

  • the if keyword;
  • one or more white spaces;
  • an expression (a question or an answer) whose value will be interpreted solely in terms of True (when its value is non-zero) and False (when it is equal to zero);
  • a colon followed by a newline;
  • an indented instruction or set of instructions (at least one instruction is absolutely required); the indentation may be achieved in two ways - by inserting a particular number of spaces (the recommendation is to use four spaces of indentation), or by using the tab character; note: if there is more than one instruction in the indented part, the indentation should be the same in all lines; even though it may look the same if you use tabs mixed with spaces, it's important to make all indentations exactly the same - Python 3 does not allow mixing spaces and tabs for indentation.

How does that statement work?

  • If the true_or_not expression represents the truth (i.e., its value is not equal to zero), the indented statement(s) will be executed;
  • if the true_or_not expression does not represent the truth (i.e., its value is equal to zero), the indented statement(s) will be omitted (ignored), and the next executed instruction will be the one after the original indentation level.

In real life, we often express a desire:

if the weather is good, we'll go for a walk
then, we'll have lunch

As you can see, having lunch is not a conditional activity and doesn't depend on the weather.

Knowing what conditions influence our behavior, and assuming that we have the parameterless functions go_for_a_walk() and have_lunch(), we can write the following snippet:

if the_weather_is_good: 
	go_for_a_walk() 

have_lunch()

Key takeaways

  1. The comparison (otherwise known as relational) operators are used to compare values. The table below illustrates how the comparison operators work, assuming that x = 0, y = 1, and z = 0:
Operator Description Example
= = returns True if operands' values are equal, and False otherwise
!= returns True if operands' values are not equal, and False otherwise x != y # True x != z # False
> True if the left operand's value is greater than the right operand's value, and False otherwise x > y # False y > z # True
< True if the left operand's value is less than the right operand's value, and False otherwise x < y # True y < z # False
True if the left operand's value is greater than or equal to the right operand's value, and False otherwise x >= y # False x >= z # True y >= z # True
True if the left operand's value is less than or equal to the right operand's value, and False otherwise x <= y # True x <= z # True y <= z # False
  1. When you want to execute some code only if a certain condition is met, you can use a conditional statement:
  • a single if statement, e.g. 
x = 10  
if x == 10: # condition  
	print("x is equal to 10") # Executed if the condition is True.
  • a series of if statements, e.g .: 
x = 10  
if x > 5: # condition one  
	print("x is greater than 5") # Executed if condition one is True.  
if x < 10: # condition two  
	print("x is less than 10") # Executed if condition two is True.  
if x == 10: # condition three  
	print("x is equal to 10") # Executed if condition three is True.

Each if statement is tested separately.

  • an if-else statement, e.g.: 
x = 10  
if x < 10: # Condition  
	print("x is less than 10") # Executed if the condition is True.  
else:  
	print("x is greater than or equal to 10") # Executed if the condition is False.
  • a series of if statements followed by an else, e.g.: 
x = 10  if x > 5: # True  
	print("x > 5")  
if x > 8: # True  
	print("x > 8")  
if x > 10: # False  
	print("x > 10")  
else:  print("else will be executed")

Each if is tested separately. The body of else is executed if the last if is False.

  • The if-elif-else statement, e.g.: 
x = 10  
if x == 10: # True  
	print("x == 10")  

if x > 15: # False  
	print("x > 15")  
	
elif x > 10: # False  
	print("x > 10")  

elif x > 5: # True  
	print("x > 5")  
else:  
	print("else will not be executed")

If the condition for if is False, the program checks the conditions of the subsequent elif blocks the first elif block that is True is executed. If all the conditions are False, the else block will be executed.

  • Nested conditional statements, e.g.: 
x = 10  
if x > 5: # True  
	if x == 6: # False  
		print("nested: x == 6")  
	elif x == 10: # True  
		print("nested: x == 10")  
	else:  
		print("nested: else")  
else:  
	print("else")

More Examples 

# Read three numbers
number1 = int(input("Enter the first number: "))
number2 = int(input("Enter the second number: "))
number3 = int(input("Enter the third number: "))

# We temporarily assume that the first number
# is the largest one.
# We will verify this soon.
largest_number = number1

# We check if the second number is larger than current largest_number
# and update largest_number if needed.
if number2 > largest_number:
    largest_number = number2

# We check if the third number is larger than current largest_number
# and update largest_number if needed.
if number3 > largest_number:
    largest_number = number3

# Print the result
print("The largest number is:", largest_number)
