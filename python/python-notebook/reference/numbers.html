Underscores in Numbers

When writing long numbers, you can regroup digits using underscores to make large numbers more readable:

universe_age = 14_000_000_000
print(universe_age)

# OUTPUT: 14000000000
  • Python ignores the underscores when storing these kinds of values, even if you don't group the digtis in threes.
    • 1000 = 1_000 = 10_00
    • works for intergers and floats

Scientific Notation

Take, for example, the speed of light, expressed in meters per second. Written directly it would look like this: 300000000.

To avoid writing out so many zeros, physics textbooks use an abbreviated form, which you have probably already seen: 3 x 108.

It reads: three times ten to the power of eight.

In Python, the same effect is achieved in a slightly different way - take a look:

print(3E8)
print(3e8)

The letter E (you can also use the lower-case letter e - it comes from the word exponent) is a concise record of the phrase times ten to the power of.

Note:

  • the exponent (the value after the E) has to be an integer;
  • the base (the value in front of the E) may be an integer.

Number Systems

#numbersystem

"In mathematics, a base or a radix is the number of different digits or combination of digits and letters that a system of counting uses to represent numbers."

For example,

  • Base 10 (Decimal)  Represent any number using 10 digits [09]
  • Base 2 (Binary) Represent any number using 2 digits [01]
  • Base 8 (Octal) Represent any number using 8 digits [07]
  • Base 16(Hexadecimal)  Represent any number using 10 digits and 6 characters [09, A, B, C, D, E, F]

Zero is very important as a place-holding value.

How to build a number system?

  1. Add another digit to the left and make the right digit 0.
  2. Then again go up to until you finish up all your symbols on the right side and when you hit the last symbol increase the digit on the left by 1.
  3. When you used up all the symbols on both the right and left digit, then make both of them 0 and add another 1 to the left and it goes on and on like that.

If you use the above 3 rules on a decimal system,

  • Write numbers 09.
  • Once you reach 9, make rightmost digit 0 and add 1 to the left which means 10.
  • Then on right digit, we go up until 9 and when we reach 19 we use 0 on the right digit and add 1 to the left, so we get 20.
  • Likewise, when we reach 99, we use 0s in both of these digits places and add 1 to the left which gives us 100.

When we have ten different symbols, when we add digits to the left side of a number, each position is going to worth 10 times more than its previous one.

Binary to ASCII

In computing, ASCII (American Standard Code for Information Interchange) is a standard encoding system for representing text using 7-bit binary numbers. In the ASCII encoding scheme, each character is represented by a unique 7-bit binary code.

To decode the binary code, we need to divide it into groups of 8 bits (1 byte) and convert each byte into its corresponding ASCII character. We can use an ASCII table or an online binary-to-text converter to perform the conversion. Here is how the decoding works:

01000001 -> 65 -> A
01101110 -> 110 -> n
01110011 -> 115 -> s
01110111 -> 119 -> w
01100101 -> 101 -> e
01110010 -> 114 -> r
00100000 -> 32 -> space
01101001 -> 105 -> i
01110011 -> 115 -> s
00100000 -> 32 -> space
00110100 -> 52 -> 4
00110010 -> 50 -> 2
00101110 -> 46 -> .

The binary code 01000001 01101110 01110011 01110111 01100101 01110010 00100000 01101001 01110011 00100000 00110100 00110010 00101110 = the ASCII text: Answer is 42.

How does ?

In the ASCII encoding scheme, each character is represented by a unique 7-bit binary code (later extended to 8 bits). The binary code 01000001 represents the decimal value 65 in the ASCII table.

To convert the binary code to the corresponding ASCII character:

  • we first need to determine the decimal value of the binary code using the positional notation of binary numbers. In positional notation, each digit of a binary number represents a power of 2, starting from the rightmost digit with a value of 2^0 (1) and increasing by a power of 2 for each digit to the left. To determine the decimal value of the binary code 01000001, we can add up the decimal values of each digit in the binary code multiplied by the corresponding power of 2: 
0 * 2^7 + 1 * 2^6 + 0 * 2^5 + 0 * 2^4 + 0 * 2^3 + 0 * 2^2 + 0 * 2^1 + 1 * 2^0 = 0 + 64 + 0 + 0 + 0 + 0 + 0 + 1 = 65`

Thus, the binary code `01000001` represents the decimal value 65, which corresponds to the ASCII character 'A'.

Octal and Hexadecimal numbers

These same rules apply to octal and hexadecimal number systems.

These same rules apply to octal and hexadecimal number systems as well. With octal, we have only 8 digits to represent numbers so once we get to 7 the next number is 10 and in hexadecimal, we have 10 digits and 6 letters to represent numbers. In that case, when we reach 9 next number is represented in the letter A. Next one B. Likewise, we go up to letter F and after F comes 10.

Octal Numbers

If an integer number is preceded by an 0O or 0o prefix (zero-o), it will be treated as an octal value. This means that the number must contain digits taken from the [0..7] range only.

0o123 is an octal number with a (decimal) value equal to 83.

The print() function does the conversion automatically

print(0o123)

The octal numeral system is the base-8 number system, and uses the digits 0 to 7. That means there are only 8 symbols or digits (0, 1, 2, 3, 4, 5, 6, 7) used to form other numbers. The main advantage of using the octal number system is that it uses fewer digits than the decimal and hexadecimal number system.

Hexadecimal Numbers

The second convention allows us to use hexadecimal numbers. Such numbers should be preceded by the prefix 0x or 0X (zero-x).

0x123 is a hexadecimal number with a (decimal) value equal to 291. The print() function can manage these values too. 

print(0x123)

Hexadecimal uses digits 09 and characters staring from A to F which altogether contains 16 different values. A single hexadecimal digit can be represented by 4 binary digits.
